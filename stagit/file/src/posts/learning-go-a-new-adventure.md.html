     16 # Why Go?
     17 
     18 As I mentioned before here on my blog, one of my goals of new years eve is to learn more about back-end development to [[Be a better Front-End Dev]].
     19 
     20 At my job, I already tried some back-end stuff with some languages:
     21 
     22 1. Node.js aka Javascript
     23 2. Python
     24 3. Java... ewww
     25 
     26 The first path that I take was using Node.js, the decision was very easy to me. I already knew [[Javascript]], so touching some AWS Lambdas and Micro-Services at my Job wasn't every hard to follow and to commit something to production.
     27 
     28 It is fine (I guess) but I wanted to learn other thing more that Javascript to expand my horizons.
     29 
     30 Later on, I started to work with Python, that is not completely new for me. In some point of my early dev career, I tried Flask to develop a simple REST API. It was very fun but not so much appealing for me to keep me in that way of back-end development.
     31 
     32 Thank god that the projects that I've been working at my job are very well designed and with good project structure. Even the test are very simple straightforward to follow. Committing a new feature for production? GG izi.
     33 
     34 Python is fun, I enjoyed PyTest so much, even more than some unit test tools in Javascript, but at the end of the day I think that I need to learn some **REAL** back-end language.
     35 
     36 > The "REAL" word that I use before it refers better to "learn a compiled language" or some "blazingly fast language". Don't be so judgy, please.
     37 
     38 So Go was on my eye for a while. I watched a lot of videos about on Youtube and gives me a lot of good impressions:
     39 
     40 1. The syntax is very well designed and elegant
     41 2. It is compiled language (blazingly fast)
     42 3. Strongly type
     43 4. Good documentation
     44 5. High employability
     45 6. Fun
     46

     51 ### How I started?
     52 
     53 I highly recommend to check to [official documentation](https://go.dev/doc/effective_go) first. It's every well written, almost well enough as the Elixir documentation _chef kiss emoji_.
     54 
     55 Next, you can GO (get it? hahaha :c ) to this next resources:
     56 
     57 1. [Go in Exercism](https://exercism.org/tracks/go): This is my first option when I need to learn some new programming language
     58 2. [Boot.dev Go Course](https://www.boot.dev/tracks/backend): Very fun and easy to follow
     59 3. [100 Go Mistakes and How to Avoid Them](https://100go.co/): This is fucking great and very useful when you want to make the things right.
     60 4. [Gophercises](https://gophercises.com/) : Imagine a gym but for learning Go, that's right.
     61 
     62 Later on I started to building something for and by myself. I'm working in a very simple CLI program to publish my Obsidian notes here on the blog and a Pokemon project... shhh is secret for now.
